Propositional Logic

The foundation of mathematical reasoning. Learn how to build, evaluate, and transform logical statements.

A proposition is a declarative statement that is either TRUE or FALSE — never both and never neither. "2+2=4" ✓ is a proposition. "Hello!" ✗ is not.

p → q is FALSE only when p is TRUE and q is FALSE. Think: "If it rains, I carry umbrella." You only break this promise if it rains AND you don't carry one.

Converse, Inverse, Contrapositive

Given p → q
Converse: q → p  |  Inverse: ¬p → ¬q  |  Contrapositive: ¬q → ¬p
KEY: A statement and its contrapositive are ALWAYS logically equivalent!

Solved Example PYQ 2025
"I go to college whenever my friends come" — find inverse and contrapositive.
1

Rewrite: "Whenever friends come → I go to college". Let p = "friends come", q = "I go to college". Statement: p → q

2

Inverse: ¬p → ¬q = "If friends do NOT come → I do NOT go to college"

3

Contrapositive: ¬q → ¬p = "If I do NOT go to college → my friends did not come"

4

Converse: q → p = "If I go to college → my friends came"

Tautology, Contradiction, Contingency

Tautology

ALWAYS true for all inputs. Final column of truth table is all T's.

Example: p ∨ ¬p

Contradiction

ALWAYS false. Final column is all F's.

Example: p ∧ ¬p

Contingency

Mix of T and F in final column. Most compound propositions fall here.

Logical Equivalence

p ≡ q means identical truth tables. Test by showing p ↔ q is a tautology.

Critical Laws to Memorize

De Morgan's Laws: ¬(p ∧ q) ≡ ¬p ∨ ¬q ¬(p ∨ q) ≡ ¬p ∧ ¬q Implication: p → q ≡ ¬p ∨ q Double Negation: ¬(¬p) ≡ p Contrapositive: p → q ≡ ¬q → ¬p Modus Ponens: p, p→q ⊢ q Modus Tollens: ¬q, p→q ⊢ ¬p

DNF and CNF Normal Forms

DNF (Disjunctive Normal Form) = OR of AND-clauses: (A∧B) ∨ (¬A∧B)
CNF (Conjunctive Normal Form) = AND of OR-clauses: (A∨B) ∧ (¬A∨B)

Solved Example — Convert to DNF
Find DNF of: (x → y) ∧ (y → x)
1

Replace implications using p→q ≡ ¬p∨q: (¬x∨y) ∧ (¬y∨x)

2

Distribute AND over OR: [(¬x∨y)∧¬y] ∨ [(¬x∨y)∧x]

3

Expand each: (¬x∧¬y) ∨ (y∧¬y) ∨ (¬x∧x) ∨ (y∧x)

4

Remove contradictions (y∧¬y = F, ¬x∧x = F):

DNF = (¬x ∧ ¬y) ∨ (x ∧ y)
Solved Example — Prove Logical Equivalence PYQ 2024
Show that ¬(p ∨ (¬p ∧ q)) ≡ ¬p ∧ ¬q using logical equivalences.
1

¬(p ∨ (¬p ∧ q))

2

Apply De Morgan's: ¬p ∧ ¬(¬p ∧ q)

3

Apply De Morgan's again: ¬p ∧ (p ∨ ¬q)

4

Distribute: (¬p ∧ p) ∨ (¬p ∧ ¬q)

5

Contradiction: F ∨ (¬p ∧ ¬q)

= ¬p ∧ ¬q ✓ Proved!

Mathematical Induction

A powerful proof technique. Like dominoes — prove the first falls, prove each knocks the next.

To prove P(n) is true for all positive integers n, you need exactly 3 steps. Missing any step = 0 marks in exam!

The 3 Steps — Follow This Exactly!

1

Base Case: Prove P(1) is true. Substitute n=1 in both LHS and RHS and verify they're equal.

2

Inductive Hypothesis: Assume P(k) is true for some arbitrary positive integer k. Write the formula with n replaced by k.

3

Inductive Step: Using the hypothesis from Step 2, prove P(k+1) is true. Write P(k+1), split out the last term, substitute P(k), then simplify to match the RHS with n=k+1.

Key exam tip: In Step 3, always start with LHS, split the last term off the sum, then use the inductive hypothesis to replace the remaining sum, then simplify algebraically.
Solved Example 1 — Classic Sum PYQ Type
Prove: 1 + 2 + 3 + ... + n = n(n+1)/2
1

Base case n=1: LHS = 1. RHS = 1(2)/2 = 1. LHS = RHS ✓

2

Inductive Hypothesis: Assume 1+2+...+k = k(k+1)/2 is true.

3

Prove for k+1: LHS = 1+2+...+k+(k+1) = k(k+1)/2 + (k+1)

4

Factor out (k+1): = (k+1)[k/2 + 1] = (k+1)(k+2)/2

5

This equals (k+1)((k+1)+1)/2 = RHS with n=k+1 ✓

Proved by induction! ∎
Solved Example 2 — Divisibility PYQ 2024
Prove n³ − n is divisible by 3 for all positive integers n.
1

Base n=1: 1³ − 1 = 0. 0 is divisible by 3 (0 = 3×0) ✓

2

IH: Assume k³ − k = 3m for some integer m (i.e., divisible by 3)

3

Compute (k+1)³ − (k+1) = k³ + 3k² + 3k + 1 − k − 1

4

= (k³ − k) + 3k² + 3k = 3m + 3k² + 3k

5

= 3(m + k² + k) which is divisible by 3 ✓

Proved! (k+1)³ − (k+1) is divisible by 3. ∎
Solved Example 3 — Odd Sum
Prove: 1 + 3 + 5 + ... + (2n−1) = n²
1

Base n=1: LHS = 1, RHS = 1² = 1 ✓

2

IH: Assume 1+3+...+(2k−1) = k²

3

For k+1: LHS = [1+3+...+(2k−1)] + (2(k+1)−1) = k² + (2k+1) = (k+1)² ✓

= (k+1)² = RHS with n=k+1 ✓ Proved! ∎

Strong Induction

In Strong Induction, the hypothesis is: "Assume P(j) is true for ALL j where 1 ≤ j ≤ k." Then prove P(k+1).

Solved Example PYQ 2025
Use strong induction to prove every integer n ≥ 2 is a product of primes.
1

Base case n=2: 2 is prime (it's a product of one prime: itself) ✓

2

Strong IH: Assume every integer j where 2 ≤ j ≤ k is a product of primes.

3

For k+1: Case 1: k+1 is prime → done (it's a product of itself).

4

Case 2: k+1 is not prime → k+1 = a×b where 2 ≤ a,b < k+1

5

By strong IH, both a and b are products of primes. So k+1 = a×b is also a product of primes ✓

Proved by strong induction! ∎

Chapter 4

Functions

Mappings between sets with strict rules. The backbone of all of mathematics.

A function f: A→B assigns EXACTLY ONE element of B to each element of A. Every element of A must be mapped. No element maps to two outputs.

Types of Functions

Injective (One-to-One)

Different inputs always give different outputs. f(a)=f(b) ⟹ a=b.

Condition: n(A) ≤ n(B) Count: n!/(n−m)!

Surjective (Onto)

Every element of B is hit. Range = Codomain. At least one preimage for every output.

Condition: n(A) ≥ n(B)

Bijective (Both)

One-to-one AND onto. Perfect pairing. ONLY bijective functions have inverses.

n(A) = n(B) Count: n!

Total Functions Count

From A (m elements) to B (n elements), total possible functions.

Total functions = nᵐ

Composition

fog(x) = f(g(x)) — apply g FIRST, then apply f to the result. Warning: fog ≠ gof in general!

Solved Example
f(x) = 2x, g(x) = x+1. Find fog(1).
1

fog(x) = f(g(x)) = f(x+1) = 2(x+1)

2

fog(1) = 2(1+1) = 4

fog(1) = 4

Inverse Function

Solved Example — Prove Abelian Group PYQ 2024 & 2025
Let G = ℝ−{0} with a * b = ab/2. Show (G,*) is an Abelian group.
1

Closure: a*b = ab/2. Since a,b ∈ ℝ−{0}, ab ≠ 0, so ab/2 ≠ 0. Thus ab/2 ∈ ℝ−{0} ✓

2

Associative: (a*b)*c = (ab/2)*c = abc/4. a*(b*c) = a*(bc/2) = abc/4. Equal ✓

3

Identity: Need a*e = a → ae/2 = a → e = 2. Check: a*2 = 2a/2 = a ✓. Identity = 2

4

Inverse: Need a*a⁻¹ = 2 (identity). a·a⁻¹/2 = 2 → a⁻¹ = 4/a. Since a≠0, 4/a ∈ ℝ−{0} ✓

5

Commutative: a*b = ab/2 = ba/2 = b*a ✓

All 5 properties hold → (G, *) is an Abelian Group ✓
Solved Example — Ring Properties PYQ 2025
In ring (R,+,·) prove: (i) a·0 = 0, (ii) a·(−b) = −(a·b), (iii) (−a)·(−b) = a·b

ii

iii

(−a)·(−b) = −(a·(−b)) [from ii] = −(−(a·b)) = a·b ✓

Order of an Element

The order of element a in group G is the smallest positive integer n such that aⁿ = e (identity element). The order of e itself is always 1.

Solved Example PYQ 2024
Find the order of each element in G = {1, −1, i, −i} under multiplication.
1

Order of 1: 1¹ = 1 = e. Order = 1

2

Order of −1: (−1)¹=−1, (−1)²=1=e. Order = 2

3

Order of i: i¹=i, i²=−1, i³=−i, i⁴=1=e. Order = 4

4

Order of −i: (−i)¹=−i, (−i)²=−1, (−i)³=i, (−i)⁴=1=e. Order = 4

Modular Arithmetic

Addition modulo m (⊕ₘ): a ⊕ₘ b = (a+b) if (a+b) < m, else (a+b) mod m
Multiplication modulo m (⊗ₘ): a ⊗ₘ b = (a×b) if (a×b) < m, else (a×b) mod m

Example: {0,1,2} with +₃
→

0⊕₃1 = 1 (1<3, keep as is) | 1⊕₃2 = 0 (3%3=0) | 2⊕₃2 = 1 (4%3=1)

Vertices, edges, and the connections between them. Used everywhere from maps to the internet.

Euler vs Hamiltonian Graphs

Euler Graph (Euler Circuit)

Traverses every EDGE exactly once and returns to start.

Condition: Graph is connected AND ALL vertices have EVEN degree.

If exactly 2 odd-degree vertices → Euler PATH (not circuit) exists.

Hamiltonian Graph

Visits every VERTEX exactly once and returns to start.

No simple necessary/sufficient condition. Must try to construct the cycle.

Dirac's theorem: If every vertex has degree ≥ n/2, Hamiltonian circuit exists.

Dijkstra's Shortest Path Algorithm

Finds the shortest path from source vertex to ALL other vertices in a weighted graph. Works only for NON-NEGATIVE edge weights.

1

Initialize: Set distance[source]=0, all others=∞. Set S={} (visited set).

2

Add source to S. Record direct distances to its neighbors.

3

Pick the unvisited vertex with smallest known distance. Add to S.

4

Update distances of its neighbors: if dist[current] + weight(edge) < dist[neighbor], update.

5

Repeat steps 3–4 until all n−1 vertices are in S.

S3

S4

Trees & Spanning Trees

Connected acyclic graphs. The most important special class of graphs.

A tree is a connected graph with NO cycles. A tree with n vertices has exactly n−1 edges. Between any two vertices there is exactly ONE path.

Binary Trees

Each node has at most 2 children (left and right).
Max nodes at height h: 2^(h+1) − 1 (perfect/full tree)
Min nodes at height h: h + 1 (straight line / degenerate tree)

Full Binary Tree

Every node has 0 OR 2 children. Never exactly 1 child.

Complete Binary Tree

All levels full except possibly last, filled left to right.

Perfect Binary Tree

All internal nodes have 2 children AND all leaves at same depth.

Array Representation

Root at index 1. Left child of i = 2i. Right child of i = 2i+1. Parent of i = ⌊i/2⌋.

Spanning Tree

A spanning tree of graph G is a subgraph that: (1) includes ALL vertices, (2) has NO cycles, (3) has exactly n−1 edges. Every connected graph has at least one.

Kruskal's Algorithm (Minimum Spanning Tree)

1

List ALL edges. Sort by weight in ASCENDING order.

2

Pick the smallest-weight edge. Does adding it create a cycle in current MST? If NO → add it. If YES → discard it.

3

Repeat until you have n−1 edges selected.

2

3

Recurrence Relations

Sequences defined by their own past values. Solve them using characteristic equations.

A recurrence relation expresses aₙ in terms of previous terms. Example: Fibonacci — aₙ = aₙ₋₁ + aₙ₋₂. We need to find a closed-form (explicit) formula for aₙ.

Homogeneous Recurrences — Characteristic Equation Method

1

Assume aₙ = rⁿ and substitute into the recurrence to get the characteristic equation.

2

Solve the characteristic equation for roots r₁, r₂.

3

If distinct roots: general solution aₙ = A·r₁ⁿ + B·r₂ⁿ

4

If repeated root r: general solution aₙ = (A + Bn)·rⁿ

5

Use initial conditions (a₀, a₁) to solve for A and B.

Solved Example — Repeated Root PYQ 2024
Solve: aₙ = 6aₙ₋₁ − 9aₙ₋₂, a₀ = 1, a₁ = 6
1

Characteristic equation: r² = 6r − 9 → r² − 6r + 9 = 0

2

Factor: (r − 3)² = 0 → r = 3 (repeated root)

3

General solution (repeated root): aₙ = (A + Bn)·3ⁿ

4

Use a₀=1: (A+0)·1 = 1 → A = 1

5

Use a₁=6: (1+B)·3 = 6 → 1+B = 2 → B = 1

aₙ = (n + 1) · 3ⁿ

Solve: aₙ = 5aₙ₋₁ − 6aₙ₋₂ + 2ⁿ + 3n

Matching coefficients: D=−3/2, E=−15/4 (solve the system)

Warshall's Algorithm — Transitive Closure

For k = 1 to n: For i = 1 to n: For j = 1 to n: M[i][j] = M[i][j] OR (M[i][k] AND M[k][j])
Solved Example PYQ 2025
Find transitive closure of R={(1,2),(1,3),(1,4),(2,3),(2,4),(3,4)} on A={1,2,3,4}.
1

Initial matrix M has 1 at positions (1,2),(1,3),(1,4),(2,3),(2,4),(3,4).

2

k=1: Check if path i→1→j exists. Rows where M[i][1]=1: none (no one maps TO 1). No change.

3

k=2: M[1][2]=1. So for all j where M[2][j]=1, set M[1][j]=1. Already 1. No change.

4

Since R is already a total order (1→2→3→4), transitive closure includes all (i,j) where i<j.

Transitive closure = {(1,2),(1,3),(1,4),(2,3),(2,4),(3,4)} — already transitive!

Generating Functions

A powerful algebraic tool for solving recurrences and counting problems.

The generating function of sequence {a₀, a₁, a₂, ...} is G(x) = a₀ + a₁x + a₂x² + ... = Σ aₙxⁿ. It "encodes" the sequence in a function we can manipulate algebraically.

Key Generating Functions to Memorize

1/(1−x) = 1 + x + x² + x³ + ... → sequence {1,1,1,1,...} 1/(1−ax) = 1 + ax + a²x² + ... → sequence {1,a,a²,a³,...} 1/(1−x)² = 1 + 2x + 3x² + 4x³ +.. → sequence {1,2,3,4,...} x/(1−x)² = x + 2x² + 3x³ + ... → sequence {0,1,2,3,...} x(1+x)/(1−x)³ = 0 + x + 4x² + 9x³+.. → sequence {0,1,4,9,...} i.e. n²
Solved Example — Solve Recurrence using GF PYQ 2024
Solve aₙ = 6aₙ₋₁ − 9aₙ₋₂, a₀=1, a₁=6 using generating functions.
1

Let G(x) = Σ aₙxⁿ. Multiply recurrence by xⁿ and sum from n=2 to ∞.

2

LHS: Σaₙxⁿ = G(x) − a₀ − a₁x = G(x) − 1 − 6x

3

RHS: 6x·(G(x) − a₀) − 9x²·G(x) = 6x·G(x) − 6x − 9x²·G(x)

4

Set LHS = RHS: G−1−6x = 6xG − 6x − 9x²G

5

Rearrange: G(1 − 6x + 9x²) = 1 → G(x) = 1/(1−3x)²

6

Expand using 1/(1−ax)² = Σ(n+1)aⁿxⁿ: G(x) = Σ(n+1)3ⁿxⁿ

aₙ = (n + 1) · 3ⁿ

Solve aₙ = 8aₙ₋₁ + 10ⁿ⁻¹, n≥1, a₁=9.

General: aₙ = C·8ⁿ + 10ⁿ/2

5

Solved Example — GF for Numeric Sequence PYQ 2024
Find the generating function for aₙ = n², n ≥ 1.
1

Start from G₁(x) = 1/(1−x) = Σxⁿ

2

Differentiate: G₁'(x) = 1/(1−x)² = Σn·xⁿ⁻¹

3

Multiply by x: x/(1−x)² = Σn·xⁿ. Call this F(x).

4

Differentiate F and multiply by x again: d/dx[x/(1−x)²] = (1+x)/(1−x)³

5

Multiply by x: G(x) = x(1+x)/(1−x)³ = Σn²xⁿ

G(x) = x(1 + x) / (1 − x)³
Chapter 10

POSET & Lattice

Ordered structures that generalize the concept of "less than or equal to".

POSET (Partially Ordered Set)

A set A with relation R is a POSET [A,R] if R is Reflexive + Antisymmetric + Transitive (remember: RAT). Written [A,≤] or [A,|] etc.

Solved Example — Prove POSET PYQ 2025
Show (P(S), ⊆) is a POSET for S = {a, b, c}. Draw its Hasse diagram.
1

Reflexive: A⊆A for all sets A ✓ (every set is a subset of itself)

2

Antisymmetric: If A⊆B and B⊆A then A=B ✓ (definition of set equality)

3

Transitive: If A⊆B and B⊆C then A⊆C ✓ (transitivity of subset relation)

4

All 3 hold → (P(S),⊆) is a POSET ✓

5

P(S) has 8 elements: ∅, {a},{b},{c}, {a,b},{a,c},{b,c}, {a,b,c}

6

Hasse: Level 0: ∅. Level 1: {a},{b},{c}. Level 2: {a,b},{a,c},{b,c}. Level 3: {a,b,c}.

7

Each set connects to sets that contain exactly one more element.

How to Draw a Hasse Diagram

1

Draw a node for each element of the set.

2

Draw an upward edge from x to y if (x,y)∈R and x≠y.

3

REMOVE all reflexive edges (self-loops). They are implied.

4

REMOVE all transitive edges. If a→b and b→c, remove a→c (it's implied by transitivity).

5

REMOVE arrowheads. Direction is always upward.

Lattice

A POSET is a Lattice if every pair of elements {a,b} has both: a Join (Least Upper Bound, a∨b) and a Meet (Greatest Lower Bound, a∧b). Both must exist and be IN the set!

In a Hasse diagram: Join = first common element ABOVE both nodes. Meet = first common element BELOW both nodes.
Solved Example — Show (P(S),⊆) is a Lattice PYQ 2025
1

For any A, B ∈ P(S): their Join = A∪B (smallest set containing both) ∈ P(S) ✓

2

Their Meet = A∩B (largest set contained in both) ∈ P(S) ✓

3

Both exist for ALL pairs → (P(S),⊆) is a Lattice ✓

Complement in Bounded Lattice

In a bounded lattice with LB (least element, bottom) and UB (greatest element, top), the complement of e is e' where:
e ∨ e' = UB (join = top) AND e ∧ e' = LB (meet = bottom)

Chapter 11

Combinatorics

Counting arrangements and selections. When order matters vs when it doesn't.

Permutation (order matters)

Arranging r items from n distinct items.

P(n,r) = n! / (n−r)!

Combination (order doesn't matter)

Selecting r items from n distinct items.

C(n,r) = n! / (r! × (n−r)!)

Sum Rule

If task A can be done m ways and task B can be done n ways (mutually exclusive), total = m + n

Product Rule

If task A (m ways) is followed by task B (n ways), total = m × n

Solved Example — Divisibility Count PYQ 2024

4

34 positive integers ≤ 100 are divisible by 4 or 9
